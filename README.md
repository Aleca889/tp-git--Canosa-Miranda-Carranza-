&#x20;Respuestas: parte 5



1). ¿Qué ventajas tiene Git frente a guardar múltiples versiones manualmente?\*\*

Git permite guardar todos los cambios de forma ordenada, volver a versiones anteriores y trabajar en equipo sin perder información. Es mucho más práctico que tener muchos archivos distintos.



2). ¿Qué funciones cumplen git add, git commit, git pull y git push?\*\*

\- git add: prepara los cambios

\- git commit: guarda los cambios con un mensaje

\- git pull: trae cambios del repositorio

\- git push: sube tus cambios



3). ¿En qué situaciones usarían ramas en un proyecto real?\*\*

Cuando se quiere agregar algo nuevo o probar cambios sin afectar la versión principal.



4). ¿Para qué usamos git rebase?\*\*

Para ordenar los cambios y mantener un historial más limpio.



5). ¿Cuándo usar merge y cuándo no?\*\*

Se usa merge para unir ramas. No se usa cuando se quiere un historial más ordenado (ahí conviene rebase).
